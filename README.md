# TikTok Draft Uploader

Личный инструмент: по одному запросу отправляет несколько видео в черновики (inbox) твоего TikTok-аккаунта через официальный **TikTok Content Posting API**.

Важно: это не обход модерации. TikTok всё равно проверяет каждый ролик в момент, когда ты сам публикуешь его из приложения — это гарантированный шаг официального режима "Upload to Inbox", и обойти его нельзя (и не нужно: именно это защищает твой аккаунт от банов за спам-загрузки).

## Что нужно сделать один раз

### 1. Зарегистрировать приложение в TikTok for Developers

1. Зайди на https://developers.tiktok.com и войди под своим TikTok-аккаунтом.
2. Создай новое приложение (App).
3. В разделе продуктов добавь **Content Posting API**.
4. В настройках приложения укажи:
   - **Redirect URI**: `http://localhost:3000/auth/callback` (или твой домен, если будешь хостить не локально)
   - **Privacy Policy URL**: ссылка на `public/privacy.html` этого проекта (нужно захостить его на любом публичном URL, например через GitHub Pages)
5. В заявке на доступ укажи сценарий использования: *"личный инструмент для планирования и загрузки собственного видео в черновики"*.
6. Дождись одобрения (обычно от нескольких дней до пары недель).

### 2. Получить ключи

После одобрения в кабинете приложения появятся:
- `Client Key`
- `Client Secret`

### 3. Настроить проект

```bash
cp .env.example .env
```

Открой `.env` и вставь свои `TIKTOK_CLIENT_KEY` и `TIKTOK_CLIENT_SECRET`.

### 4. Установить зависимости и запустить

```bash
npm install
npm start
```

Сервер поднимется на http://localhost:3000

### 5. Авторизоваться

Открой http://localhost:3000, нажми **"Войти через TikTok"**, разреши доступ. Токены сохранятся в `data/tokens.json` и будут обновляться автоматически.

## Как пользоваться

1. Открой http://localhost:3000
2. Перетащи или выбери несколько видео
3. Нажми **"Отправить в черновики"**
4. Дождись, пока каждое видео получит статус "отправлено"
5. Открой приложение TikTok на телефоне — в inbox появятся уведомления по каждому видео. Опубликовать каждое всё равно нужно вручную из приложения — так работает официальный режим черновиков.

## Ограничения, которые стоит знать

- TikTok ограничивает инициализацию загрузки 6 запросами в минуту на аккаунт — поэтому между видео есть пауза в несколько секунд, это нормально.
- Один прогон поддерживает до 20 видео за раз (легко увеличить в `server.js`, параметр `upload.array('videos', 20)`).
- Формат видео должен соответствовать требованиям TikTok (MP4/WebM, до 4 ГБ, 3–600 секунд).
- Режим Upload to Inbox не поддерживает добавление фирменных звуков TikTok, стикеров и эффектов — их можно добавить только вручную в приложении при публикации.

## Если что-то меняется в API

TikTok периодически обновляет Content Posting API. Актуальная документация: https://developers.tiktok.com/doc/content-posting-api-get-started-overview
